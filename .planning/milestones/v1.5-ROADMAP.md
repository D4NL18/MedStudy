# Milestone v1.5: Planos de Usuário e Monetização (PIX)

**Status:** ✅ SHIPPED 2026-07-16
**Phases:** 40-43
**Total Plans:** 4

## Overview

Implementado o paywall com trial de 30 dias grátis, assinatura anual via PIX (API Banco do Brasil), painel admin para concessão manual de acessos e notificações de expiração.

## Phases

### Phase 40: Modelo de Assinaturas & Motor de Trial Freemium no Backend

**Goal**: Implementar paywall com trial de 30 dias
**Plans**: 1 plan

**Details:**
Entidades JPA (`Subscription`, `PixTransaction`), concessão automática de 30 dias de trial no cadastro, scheduler para expiração e interceptor de segurança para bloqueio de contas expiradas.

### Phase 41: Integração API PIX Banco do Brasil & Webhooks (Spring Boot)

**Goal**: Integração de PIX dinâmico
**Plans**: 1 plan

**Details:**
Cliente OAuth2 + mTLS para API BB PIX, endpoints REST de geração de PIX Dinâmico, webhook receiver seguro e mecanismo de fallback ("Já Paguei") com suporte a profile dev `mock-pix`.

### Phase 42: Painel Administrativo de Assinaturas (Single Admin)

**Goal**: Painel para gerenciamento de usuários
**Plans**: 1 plan

**Details:**
Controle de acessos por Role `ADMIN`, endpoints e tela no Angular para gerenciar usuários, conceder/estender acesso manual (VIP/Lifetime) e visualizar relatórios de pagamentos.

### Phase 43: Interface do Usuário (Angular NgRx), Tela de Planos & UX Polish

**Goal**: Interface de monetização
**Plans**: 1 plan

**Details:**
Tela de Planos (`/planos`), modal de checkout PIX com QR Code e Copia e Cola, NgRx State slice de assinatura, Route Guard (`CanActivateFn`), banners de aviso de expiração e histórico de pagamentos na conta.

## Milestone Summary

**Key Decisions:**
- Decision: Utilização de polling no frontend via `ChangeDetectorRef` e mecanismo de Refresh subject para detecção de mudança de status da transação em tempo real.
- Decision: Permitir mock no webhook local para simular pagamentos durante desenvolvimento.

**Issues Resolved:**
- Tratamento da exceção 403 Forbidden no `JwtAuthenticationFilter` durante o registro de usuário antes do JWT estar disponível.
- Ajustes no painel admin para visualização correta de dados e overflow da tabela.

- N/A
