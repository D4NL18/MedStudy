# Requirements: MedStudy — Plataforma de Estudos Médicos

**Defined:** 2026-05-05
**Core Value:** O estudante registra desempenho em questões, acompanha revisões e vê sua evolução — com dados seguros, rápidos e confiáveis.

---

## v1 Requirements

### Authentication

- [x] **AUTH-01**: Usuário pode fazer login com e-mail e senha
- [x] **AUTH-02**: Usuário pode solicitar recuperação de senha via e-mail
- [x] **AUTH-03**: Backend emite JWT access token (curto prazo) + refresh token (longo prazo)
- [x] **AUTH-04**: Sessão persiste após reload do browser via refresh token armazenado de forma segura
- [x] **AUTH-05**: Rotas protegidas retornam 401 sem token válido
- [x] **AUTH-06**: Refresh token rotation — token antigo invalidado após uso
- [x] **AUTH-07**: Logout invalida o refresh token no servidor

### Infrastructure

- [x] **INFR-01**: Monorepo com diretórios `frontend/` (Angular) e `backend/` (Spring Boot) na raiz
- [x] **INFR-02**: PostgreSQL 16 disponível localmente (via Docker Compose)
- [x] **INFR-03**: Flyway para versionamento e execução de migrations SQL
- [x] **INFR-04**: Swagger/OpenAPI 3 acessível em `/api/docs`
- [x] **INFR-05**: `.env` / `application.properties` para variáveis sensíveis (sem secrets no código)
- [x] **INFR-06**: Scripts `npm run dev` e `./mvnw spring-boot:run` funcionam out-of-the-box

### Backend Core

- [x] **BACK-01**: Estrutura em camadas: Controller → Service → Repository (nenhuma lógica de negócio em Controller)
- [x] **BACK-02**: DTOs com validação `jakarta.validation` para todas as entradas (request bodies)
- [x] **BACK-03**: MapStruct para conversão bidirecional Entity ↔ DTO
- [x] **BACK-04**: `@ControllerAdvice` com handlers para todas as exceções customizadas
- [x] **BACK-05**: Respostas de erro não expõem stack trace nem informações internas
- [x] **BACK-06**: Paginação obrigatória em todos os endpoints de listagem (máximo configurável, default 20)
- [x] **BACK-07**: CORS configurado estritamente (apenas origin do frontend permitida)

### Security

- [x] **SECU-01**: Spring Security com filtro JWT em todas as rotas (exceto `/api/auth/**`)
- [x] **SECU-02**: Senhas armazenadas com BCrypt (fator de custo ≥ 10)
- [x] **SECU-03**: Proteção contra SQL Injection via JPA/Hibernate (queries parametrizadas)
- [x] **SECU-04**: Headers de segurança HTTP configurados (X-Frame-Options, X-Content-Type-Options, etc.)
- [x] **SECU-05**: Limite de payload de requisição configurado (max body size)
- [x] **SECU-06**: Rate limiting básico no endpoint de login (prevenção de brute-force)
- [x] **SECU-07**: Tokens JWT com expiração adequada (access: 15min, refresh: 7 dias)

### Dashboard

- [x] **DASH-01**: Exibe taxa global de acertos (% corretas / total de questões)
- [x] **DASH-02**: Exibe total de questões no mês atual e no ano atual
- [x] **DASH-03**: Identifica e exibe a Grande Área mais forte (maior % acerto)
- [x] **DASH-04**: Identifica e exibe a Grande Área mais fraca (menor % acerto)
- [x] **DASH-05**: Exibe progresso teórico (% de aulas assistidas sobre total)
- [x] **DASH-06**: Exibe streak (sequência de dias consecutivos com sessão de estudo)
- [x] **DASH-07**: Gráfico de linha — evolução mensal de % acertos nos 12 meses do ano
- [x] **DASH-08**: Gráfico de barras horizontais — % acerto por Grande Área (colorização por faixa)
- [x] **DASH-09**: Gráfico de pizza — distribuição de volume de questões por área
- [x] **DASH-10**: Lista de top 5 aulas com prioridade Alta ou Diamante não assistidas

### Banco de Questões (Sessões de Estudo)

- [x] **BNCO-01**: Criar sessão de estudo (grande_area, tema, data, qts_feitas, qts_corretas, instituicao, data_proxima_revisao)
- [x] **BNCO-02**: Editar sessão de estudo existente
- [x] **BNCO-03**: Excluir sessão com modal de confirmação
- [x] **BNCO-04**: Listar sessões com paginação e ordenação por qualquer coluna
- [x] **BNCO-05**: Filtrar por grande_area, faixa de desempenho (<70%, 70–80%, >80%) e status de revisão
- [x] **BNCO-06**: Busca textual por tema, grande_area e instituição
- [x] **BNCO-07**: Cards de métricas: total de sessões, total de questões, média de acertos, revisões críticas pendentes

### Simulados

- [x] **SIML-01**: Criar simulado com nome, data e dados por área (CM, Cir, Ped, GO, Prev): total, acertos, erros
- [x] **SIML-02**: Calcular automaticamente erros = total − acertos (validação server-side)
- [x] **SIML-03**: Editar simulado existente
- [x] **SIML-04**: Excluir simulado com confirmação
- [x] **SIML-05**: Listar histórico de simulados com métricas agregadas (total geral, % acerto global)
- [x] **SIML-06**: Visualizar detalhes de simulado com breakdown por área

### Plano de Aulas

- [x] **PLAN-01**: Criar entrada de aula (grande_area, tema, prioridade: Diamante/Alta/Média/Baixa, aula_assistida: boolean)
- [x] **PLAN-02**: Editar aula existente
- [x] **PLAN-03**: Excluir aula com confirmação
- [x] **PLAN-04**: Marcar/desmarcar aula como assistida
- [x] **PLAN-05**: Filtrar por grande_area, prioridade e status (assistida/não assistida)
- [x] **PLAN-06**: Busca textual por tema

### Análise por Área

- [x] **AREA-01**: Agregar estatísticas de desempenho por grande_area (total de questões, % acerto, sessões)
- [x] **AREA-02**: Gráfico comparativo de % acerto entre as 5 grandes áreas
- [x] **AREA-03**: Ranking de áreas por desempenho (melhor → pior)

### Análise por Tema

- [x] **TEMA-01**: Agregar estatísticas de desempenho por tema dentro de cada grande_area
- [x] **TEMA-02**: Tabela com colunas: tema, área, total de questões, % acerto, nº de sessões
- [x] **TEMA-03**: Ordenação e filtragem da tabela por qualquer coluna

### Revisão Intervalada

- [x] **REVI-01**: Listar sessões com revisão atrasada (data_proxima_revisao < hoje, não concluída)
- [x] **REVI-02**: Listar sessões para revisar hoje
- [x] **REVI-03**: Listar revisões futuras agendadas
- [x] **REVI-04**: Listar revisões já concluídas
- [x] **REVI-05**: Marcar revisão como concluída (atualiza revisao_concluida = true)
- [x] **REVI-06**: Desfazer marcação de revisão (revisao_concluida = false)
- [x] **REVI-07**: Cards de contagem por categoria (atrasadas, hoje, futuras, realizadas)

### Flashcards

- [x] **FLSH-01**: Criar flashcard (grande_area, frente JSONB, verso JSONB)
- [x] **FLSH-02**: Editar flashcard existente
- [x] **FLSH-03**: Excluir flashcard com confirmação
- [x] **FLSH-04**: Modo de estudo: exibir frente → flip → exibir verso
- [x] **FLSH-05**: Avaliar dificuldade após revisão: Fácil / Médio / Difícil
- [x] **FLSH-06**: Calcular próxima data de revisão baseada na dificuldade (Fácil: +7d, Médio: +3d, Difícil: +1d)
- [x] **FLSH-07**: Filtrar flashcards por grande_area
- [x] **FLSH-08**: Exibir contagem de flashcards por estado (para revisar hoje, atrasados, em dia)

### Theming (Sistema de Temas de Cores)

- [x] **THEM-01**: Sistema de temas baseado em CSS Custom Properties (variáveis) — nenhum valor de cor hardcoded nos components
- [x] **THEM-02**: Tema **Rosa** (padrão) — paleta do legado (`#430428`, `#6F0642`, `#F553B0`, `#FBBCE0`)
- [x] **THEM-03**: Tema **Claro** — base branca/cinza clara, acentos neutros e profissionais
- [x] **THEM-04**: Tema **Escuro** — dark mode com fundo `#0f1117`, superfícies `#1a1d27`, acentos em índigo/violeta
- [x] **THEM-05**: Tema **Verde** — tons de esmeralda/floresta (`#065f46`, `#10b981`), remetendo à área médica
- [x] **THEM-06**: Tema **Azul** — azul cobalto/marinho (`#1e40af`, `#3b82f6`), estilo clínico clássico
- [x] **THEM-07**: Tema **Vermelho** — vermelho cardinal (`#991b1b`, `#ef4444`), tom de urgência/energia
- [x] **THEM-08**: Tema **Roxo** — violeta/lavanda (`#4c1d95`, `#8b5cf6`), estilo moderno e distinto
- [x] **THEM-09**: Tema **Laranja** — âmbar/caramelo (`#92400e`, `#f59e0b`), tom caloroso e motivacional
- [x] **THEM-10**: Seletor de tema acessível na barra de navegação (ícone de paleta + dropdown/modal)
- [x] **THEM-11**: Preferência de tema persistida em `localStorage` — mantida após reload e entre sessões
- [x] **THEM-12**: Todos os gráficos (linha, barras, pizza) adaptam suas cores primárias ao tema ativo
- [x] **THEM-13**: Transição suave ao trocar de tema (CSS `transition` nas variáveis — `200ms ease`)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Aplicativo mobile nativo | Web-first; alta complexidade; v2+ |
| Multi-usuário / multi-tenant | Instância pessoal na v1; auth isola por usuário |
| Upload de PDFs e imagens de questões | Risco de segurança, storage complexo; v2+ |
| Integração com bancos de questões externos | API licensing; fora do escopo inicial |
| Deploy em cloud (AWS, GCP, Azure) | Ambiente local na v1; v2+ |
| OAuth2/SSO (Google, etc.) | Email/senha suficiente para v1 |
| Chat ou comunidade | Não é core value |
