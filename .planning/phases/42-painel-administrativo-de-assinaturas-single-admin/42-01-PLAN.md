# Phase 42 Plan 01 — Painel Administrativo de Assinaturas (Single Admin)

## Goal
Implementar os endpoints backend de administração protegidos por `@PreAuthorize("hasRole('ADMIN')")` e a interface no Angular para gerenciamento de assinaturas, concessão/extensão manual de plano com justificativa, estatísticas operacionais e financeiras, e histórico de transações PIX.

---

<task id="42-01-02" wave="1">
<title>Criar AdminSubscriptionService, AdminSubscriptionController e Testes Unitários (ADMIN-01, ADMIN-02, ADMIN-03)</title>
<read_first>
  - backend/src/main/java/com/medstudy/backend/modules/subscription/service/PixPaymentService.java
  - backend/src/main/java/com/medstudy/backend/core/config/SecurityConfig.java
</read_first>
<action>
1. Criar `AdminSubscriptionService` em `com.medstudy.backend.modules.subscription.service`:
   - Método `AdminSubscriptionStatsDto getStats()`: calcula contagens por status e total de faturamento via repository.
   - Método `Page<AdminUserSubscriptionDto> listUserSubscriptions(String search, SubscriptionStatus status, Pageable pageable)`: retorna lista paginada.
   - Método `AdminUserSubscriptionDto overrideUserSubscription(UUID userId, AdminOverrideRequestDto request)`:
     - Busca `Subscription` do usuário.
     - Valida obrigatoriedade de `notes` (não pode ser vazio/nulo).
     - Aplica alteração:
       - `ADD_30_DAYS`: base = `currentPeriodEnd` no futuro ? `currentPeriodEnd` : `now`; `newEnd = base + 30 dias`; `status = ACTIVE`.
       - `ADD_90_DAYS`: `newEnd = base + 90 dias`; `status = ACTIVE`.
       - `ADD_365_DAYS`: `newEnd = base + 365 dias`; `status = ACTIVE`.
       - `GRANT_LIFETIME`: `status = LIFETIME`, `currentPeriodEnd = null`.
       - `FORCE_EXPIRE`: `status = EXPIRED`, `currentPeriodEnd = now`.
     - Seta `isAdminOverride = true`, `notes = request.notes()`.
     - Salva no banco de dados.
     - Evicta cache Caffeine `subscriptionStatusCache` via `cacheManager.getCache(...).evict(userId)`.
   - Método `Page<AdminPixTransactionDto> listPixTransactions(PixStatus status, Pageable pageable)`: retorna lista paginada de cobranças PIX.
2. Criar `AdminSubscriptionController` em `com.medstudy.backend.modules.subscription.controller` anotado com `@RestController`, `@RequestMapping("/api/admin/subscriptions")` e `@PreAuthorize("hasRole('ADMIN')")`:
   - `GET /stats`: chama `adminSubscriptionService.getStats()`.
   - `GET /users`: chama `listUserSubscriptions(search, status, pageable)`.
   - `POST /users/{userId}/override`: chama `overrideUserSubscription(userId, request)`.
   - `GET /transactions`: chama `listPixTransactions(status, pageable)`.
3. Criar testes unitários:
   - `AdminSubscriptionServiceTest.java` (valida os cálculos e overrides).
   - `AdminSubscriptionControllerTest.java` (valida a trava `@PreAuthorize("hasRole('ADMIN')")`).
</action>
<acceptance_criteria>
  - `./mvnw test -Dtest=AdminSubscriptionServiceTest,AdminSubscriptionControllerTest` passa 100%
  - `@PreAuthorize("hasRole('ADMIN')")` bloqueia acesso de não-admins
</acceptance_criteria>
</task>
