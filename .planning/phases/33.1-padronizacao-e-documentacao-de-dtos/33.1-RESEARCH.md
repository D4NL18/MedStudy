## 1. Domain & Scope
The objective is to refactor all backend DTOs to standard Java classes using Lombok, centralize annotation messages, and add exhaustive Swagger documentation.
Requirements:
- **BKND-03**: Guarantee all DTOs are classes (not records), using Lombok to avoid boilerplate. Centralize annotation messages in a single variables file.
- **DOC-03**: Ensure Swagger documentation inside DTOs is as complete as possible (description and example for all fields).

## 2. Constraints & Decisions (from 33.1-CONTEXT.md)
- **Message Structure**: Create `ValidationMessages` interface/class in `com.medstudy.backend.core.constants`.
- **Lombok**: Use `@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor` on all DTO classes instead of pure Java boilerplate.
- **Swagger**: `@Schema` must have both `description` and `example` for ALL attributes.

## 3. Implementation Plan Focus
- **Step 1:** Create `com.medstudy.backend.core.constants.ValidationMessages`.
- **Step 2:** Refactor existing backend DTOs (e.g., in `com.medstudy.backend.dto.*`) to replace any `record` declarations with standard `class` declarations + Lombok annotations.
- **Step 3:** Move any hardcoded validation messages (e.g., in `@NotNull(message = "...")`) to `ValidationMessages`.
- **Step 4:** Add `@Schema(description = "...", example = "...")` to all DTO attributes. All descriptions must be in English.

## 4. Validation Architecture
- Run `mvn compile` to ensure Lombok annotations process correctly.
- Run `npm run lint` or `mvn checkstyle:check` to ensure no syntax issues.
- Start the application and verify the Swagger UI at `/swagger-ui.html` properly displays descriptions and examples for all DTO schemas.
