# Phase 40 Plan 01 — Modelo de Assinaturas, Trial de 30 Dias e Filtro de Segurança Freemium

## Goal
Implementar o modelo de dados de assinaturas (`Subscription` e `PixTransaction`), a concessão automática de 30 dias de trial gratuito ao cadastrar usuários, o agendador de expiração em background (`@Scheduled`) e o filtro de segurança do Spring Security (`SubscriptionStatusFilter`) com cache em memória (Caffeine) e resposta HTTP 402 Payment Required para contas expiradas.

---

<task id="40-01-04" wave="2">
<title>Implementar SubscriptionStatusFilter e Interceptador de Bloqueio no Spring Security (PAYWALL-02)</title>
<read_first>
  - backend/src/main/java/com/medstudy/backend/core/config/SecurityConfig.java
  - backend/src/main/java/com/medstudy/backend/core/security/JwtAuthenticationFilter.java
  - .planning/phases/40-modelo-de-assinaturas-motor-de-trial-freemium-no-backend/40-CONTEXT.md
</read_first>
<action>
1. Criar a classe `SubscriptionStatusFilter` no pacote `com.medstudy.backend.core.security` estendendo `OncePerRequestFilter`:
   - Injetar `SubscriptionRepository` e `CacheManager`.
   - Sobrescrever `shouldNotFilter(HttpServletRequest request)` para liberar requisições cujos caminhos iniciem por:
     - `/api/v1/auth/`
     - `/api/v1/subscriptions/`
     - `/swagger-ui/`, `/v3/api-docs/`
   - No método `doFilterInternal`:
     - Obter o usuário autenticado no `SecurityContextHolder.getContext().getAuthentication()`. Se não autenticado, apenas repassar `filterChain.doFilter(request, response)`.
     - Obter o ID do usuário e consultar o status de assinatura com cache em memória.
     - Se o status for `EXPIRED` (ou `trialEndDate`/`currentPeriodEnd` passados), interceptar a requisição:
       - Definir `response.setStatus(402)` (HTTP Status `HttpServletResponse.SC_PAYMENT_REQUIRED`).
       - Definir `response.setContentType("application/json;charset=UTF-8")`.
       - Escrever o corpo JSON estruturado:
         ```json
         {
           "code": "PAYWALL_REQUIRED",
           "message": "Assinatura ou período de teste expirado",
           "expiredAt": "<ISO_8601_TIMESTAMP>",
           "status": "EXPIRED"
         }
         ```
       - Retornar sem chamar `filterChain.doFilter()`.
2. Registrar o filtro em `SecurityConfig.java`:
   - Injetar `SubscriptionStatusFilter` e adicioná-lo à cadeia de filtros usando `.addFilterAfter(subscriptionStatusFilter, JwtAuthenticationFilter.class)`.
3. Criar os testes unitários em `SubscriptionStatusFilterTest.java` validando bypass para rotas isentas, liberação para contas válidas e emissão do JSON 402 para contas expiradas.
</action>
<acceptance_criteria>
  - `./mvnw test -Dtest=SubscriptionStatusFilterTest` executa com sucesso e passa todos os cenários
  - `SubscriptionStatusFilter.java` intercepta requisições escrevendo o status 402 com o código `PAYWALL_REQUIRED`
  - `SecurityConfig.java` contém `.addFilterAfter(subscriptionStatusFilter, JwtAuthenticationFilter.class)`
</acceptance_criteria>
</task>
