# Phase 41 Plan 01 — Integração API PIX Banco do Brasil & Webhooks (Spring Boot)

## Goal
Implementar a integração de pagamentos PIX com o Banco do Brasil (OAuth2, mTLS, cobrança dinâmica), receptáculo público de Webhook com idenpotência, mecanismo de verificação manual ("Já Paguei") com rate-limit e suporte ao profile de simulação dev `mock-pix`.

<task id="41-01-02" wave="1">
<title>Criar PixPaymentService e Receptáculo de Webhooks (PixWebhookController) (PIX-02)</title>
<read_first>
  - backend/src/main/java/com/medstudy/backend/modules/subscription/repository/PixTransactionRepository.java
  - backend/src/main/java/com/medstudy/backend/modules/subscription/repository/SubscriptionRepository.java
  - backend/src/main/java/com/medstudy/backend/core/config/CacheConfig.java
</read_first>
<action>
1. Criar o serviço `PixPaymentService` em `com.medstudy.backend.modules.subscription.service`:
   - Injetar `PixTransactionRepository`, `SubscriptionRepository`, `PixClient` e `CacheManager`.
   - Método `PixResponseDto generatePixCharge(User user)`: cria a cobrança com 15 minutos (900s) de expiração, salva `PixTransaction` com status `CREATED` e retorna o DTO.
   - Método `boolean processPaymentSuccess(String txid, String endToEndId, Instant paidAt)`:
     - Busca `PixTransaction` pelo `txid`.
     - Verifica idempotência (se já estiver `PAID`, retorna `true` imediatamente).
     - Atualiza `PixTransaction`: `status = PAID`, `endToEndId = endToEndId`, `paidAt = paidAt`.
     - Atualiza `Subscription` do usuário:
       - Se `trialEndDate` ou `currentPeriodEnd` no futuro, estende `currentPeriodEnd` por +365 dias a partir da data de vencimento atual.
       - Se expirado, `currentPeriodEnd = now.plus(365, DAYS)`.
       - Atualiza `status = ACTIVE`.
     - Evicta o cache de status do usuário via `cacheManager.getCache(CacheConfig.SUBSCRIPTION_STATUS_CACHE).evict(userId)`.
2. Criar `PixWebhookController` no pacote `com.medstudy.backend.modules.subscription.controller`:
   - Endpoint `POST /api/webhooks/pix/bb`.
   - Processa os itens do `BbWebhookPayloadDto`, invoca `pixPaymentService.processPaymentSuccess(pix.getTxid(), pix.getEndToEndId(), ...)` e retorna `HTTP 200 OK`.
3. Criar o teste unitário `PixWebhookControllerTest.java` validando o recebimento e ativação via webhook.
</action>
<acceptance_criteria>
  - `./mvnw test -Dtest=PixWebhookControllerTest` executa com sucesso
  - `PixPaymentService` estende a validade da assinatura em +365 dias sem perder o saldo de trial/período vigente
  - `cacheManager.getCache(...).evict(userId)` é chamado na ativação
</acceptance_criteria>
</task>

<task id="41-01-03" wave="2">
<title>Criar SubscriptionPixController (Geração de PIX e Fallback "Já Paguei") (PIX-01, PIX-03)</title>
<read_first>
  - backend/src/main/java/com/medstudy/backend/modules/subscription/service/PixPaymentService.java
  - backend/src/main/java/com/medstudy/backend/core/config/SecurityConfig.java
</read_first>
<action>
1. Criar `SubscriptionPixController` em `com.medstudy.backend.modules.subscription.controller` mapeado em `@RequestMapping("/api/subscriptions/pix")`:
   - Endpoint `POST /api/subscriptions/pix/create`:
     - Requer autenticação (JWT).
     - Chama `pixPaymentService.generatePixCharge(currentUser)` e retorna HTTP 201 Created com `PixResponseDto`.
   - Endpoint `GET /api/subscriptions/pix/{txid}/status`:
     - Requer autenticação (JWT).
     - Implementa verificação manual do botão "Já Paguei":
       - Consulta banco de dados local primeiro.
       - Se o status local for `CREATED`, verifica se o último `updatedAt` ocorreu há pelo menos 5 segundos (rate-limit).
       - Se permitido, consulta a API do cliente PIX (`pixClient.getChargeStatus(txid)`).
       - Se a API indicar concluída, processa a ativação da assinatura.
       - Retorna `PixStatusResponseDto` com o status atualizado.
</action>
<acceptance_criteria>
  - `POST /api/subscriptions/pix/create` retorna 201 Created com os dados do PIX
  - `GET /api/subscriptions/pix/{txid}/status` respeita o rate-limit de 5s antes de consultar o gateway remoto
</acceptance_criteria>
</task>
